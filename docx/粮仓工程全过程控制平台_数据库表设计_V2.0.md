# 粮仓工程全过程控制平台 — 数据库表设计 V2.1

> **基于**: 数据字典草案（详细版）V2.0 + 数据库设计规范 V1.1 + 前端需求文档（回答版）  
> **目标数据库**: PostgreSQL 16+  
> **日期**: 2026-07-06  
> **金额单位**: 数据库统一存储 **分**（BIGINT），源数据导入时需转换（万元→×1000000，元→×100）  
> **主键策略**: UUID，默认 `gen_random_uuid()`  
> **时区**: 全部 `TIMESTAMPTZ`，`DATE` 无时区

## 二、完整建表 SQL

> **说明**: 以下 SQL 遵循 V1.0 设计规范的所有约定（UUID PK、BIGINT 金额=分、TIMESTAMPTZ、软删除、updated_at 触发器）。字典表和外键约束在第三节单独列出。

### 2.1 项目主表 `t_projects`

**来源**: 文件5（新投+续投）、文件6（目标计划）、文件7（省市重点）

```sql
CREATE TABLE t_projects (
    id              UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    project_name    VARCHAR(200)    NOT NULL,                    -- 项目名称（如"川西现代粮食物流中心建设项目"）
    project_code    VARCHAR(50)     UNIQUE NOT NULL,             -- 项目编码（业务唯一标识，如 XM-2025-001）
    -- 项目分类属性（来自文件5 新投/续投）
    project_nature  VARCHAR(50)     NOT NULL DEFAULT '功能性项目', -- 项目性质：功能性项目/商业性项目
    invest_timing   VARCHAR(20)     NOT NULL DEFAULT '续投项目',   -- 投资时序：新投项目/续投项目
    invest_nature   VARCHAR(50),                                 -- 投资性质：固定资产投资/股权投资/其他投资
    invest_structure VARCHAR(50),                                -- 投资结构：基础设施类/民生和社会事业类/生态环保类
    invest_field    VARCHAR(100),                                -- 投资领域：装备制造/食品饮料/材料化工/...
    implement_body  VARCHAR(200),                                -- 实施主体（如"产投集团"，可能含换行拼接）
    implement_period VARCHAR(50),                                -- 实施时间（如"2025.12-2027.11"）
    expected_return TEXT,                                        -- 投资预期收益说明
    business_class  VARCHAR(20),                                 -- 业务划分：主业/非主业/培育业
    region          VARCHAR(20)  DEFAULT '市内',                  -- 区域：市内/市外
    -- 项目行政属性（来自文件6）
    importance      VARCHAR(10),                                 -- 重要性：★(省重点)/▲(市重点)
    supervising_dept VARCHAR(200),                               -- 行业归口主管部门（如"德阳市发展与改革局"）
    project_level_remark VARCHAR(100),                           -- 项目级别备注（省重点项目/子项标注）
    responsible_unit VARCHAR(200),                               -- 责任单位（如"罗江区人民政府"）
    -- 建设内容与规模
    construction_content TEXT,                                   -- 项目内容/建设内容（来自文件5）
    construction_scale  TEXT,                                    -- 建设内容及规模（来自文件7）
    construction_period VARCHAR(50),                             -- 建设起止年（如"2025-2027年"）/建设周期（如"600天"）
    -- 投资金额（单位：分，导入时万元×1000000）
    planned_total_invest    BIGINT  NOT NULL DEFAULT 0,          -- 计划总投资
    planned_invest_2026     BIGINT  DEFAULT 0,                   -- 2026年计划投资
    -- 质量安全目标（来自文件6）
    quality_target   VARCHAR(50)  NOT NULL DEFAULT '合格',             -- 质量目标
    safety_target    VARCHAR(100),                               -- 安全目标（如"不发生一般及以上的安全事故"）
    -- 参建单位
    owner_unit       VARCHAR(200),                               -- 业主单位/建设单位
    -- 项目状态
    project_status   VARCHAR(20)  NOT NULL DEFAULT 'preparing',  -- preparing/constructing/completed/suspended
    -- 时间戳
    created_at       TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at       TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    deleted_at       TIMESTAMPTZ                                -- 软删除
);

COMMENT ON TABLE t_projects IS '项目主表，整合产投月报（文件5）、目标计划表（文件6）、省市重点项目进度（文件7）的项目元数据';
COMMENT ON COLUMN t_projects.planned_total_invest IS '计划总投资，单位：分（数据库内部），前端展示÷1000000=万元';
```

### 2.2 合同主表 `t_contracts`

**来源**: 文件1（合同支付情况汇总表.xlsx → 合同支付情况 Sheet，Row 2-3 表头）

```sql
CREATE TABLE t_contracts (
    id              UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id      UUID            NOT NULL,                    -- FK → t_projects
    -- 合同业务字段（对应 Excel 列 A~F）
    seq             INTEGER         NOT NULL,                    -- 序号（Row 内的行号）
    contract_no     VARCHAR(20)     UNIQUE NOT NULL,             -- 编号（CX001~CX018，业务主键）
    supplier_name   VARCHAR(200)    NOT NULL,                    -- 供应商/承包单位
    contract_desc   VARCHAR(500)    NOT NULL,                    -- 合同内容（合同类型/内容描述）
    sign_date       DATE,                                       -- 签订日期
    contract_amount BIGINT          DEFAULT 0,                   -- 合同金额（分）。⚠ 源数据此列多数为空，待补填
    -- 合同分类
    contract_type   VARCHAR(30)     NOT NULL DEFAULT 'secondary',-- main(主合同)/secondary(二类费用)/supplementary(补充)
    -- 状态
    status          VARCHAR(20)     NOT NULL DEFAULT 'active',   -- active/expired/terminated
    -- 时间戳
    created_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    deleted_at      TIMESTAMPTZ
);

COMMENT ON TABLE t_contracts IS '合同主表，数据来源于二类费用台账/合同支付情况汇总表.xlsx';
COMMENT ON COLUMN t_contracts.contract_no IS '合同唯一编号，格式 CX001~CX018，全局唯一';
COMMENT ON COLUMN t_contracts.contract_amount IS '⚠ 源数据中 Col F(合同金额) 数据行全为空，需业务人员补填。单位：分';
```

### 2.3 合同支付阶段表 `t_contract_payment_stages`

**来源**: 文件1（合同支付情况 Sheet，G~K 列 + L~O 列汇总数据）  
**设计思路**: 不同合同的付款阶段数不同（3~5 个），不适合硬编码固定列，故设计为子表（每个阶段一行）。

```sql
CREATE TABLE t_contract_payment_stages (
    id              UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    contract_id     UUID            NOT NULL,                    -- FK → t_contracts
    contract_no     VARCHAR(20)     NOT NULL,                    -- 冗余 contract_no，方便直接查询
    -- 支付阶段
    stage_order     INTEGER         NOT NULL,                    -- 阶段序号（1=施工前/预付款, 2=施工中, 3=竣工, 4=结算后, 5=质保尾款）
    stage_name      VARCHAR(50)     NOT NULL,                    -- 阶段名称（施工前阶段/施工中阶段/竣工阶段/尾款阶段）
    payment_terms   TEXT,                                       -- 支付条款原文（如"合同签订后30日内，支付20%预付款"）
    -- 汇总数据（每个合同仅一行，冗余存储于 stage_order=0 或独立查询）
    cumulative_paid     BIGINT      DEFAULT 0,                   -- 累计已付（分）
    remaining_unpaid    BIGINT      DEFAULT 0,                   -- 剩余未付（分）
    paid_ratio          DECIMAL(8,4),                            -- 已付比例（可能为 NULL，源数据有 #DIV/0!）
    remaining_ratio     DECIMAL(8,4),                            -- 剩余未付比例
    -- 时间戳
    created_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW()
);

COMMENT ON TABLE t_contract_payment_stages IS '合同支付阶段+汇总数据，来源于合同台账 G~O 列。阶段行(stage_order≥1)存支付条款，汇总行(stage_order=0)存累计/剩余金额';
```

> **设计说明**: 支付阶段示例数据 —  
> CX005（中储粮，设计合同）3 阶段: ①施工前=支付20%预付款, ②施工中=支付至50%, ③竣工=支付至80%, ④尾款=竣工财评后支付  
> CX008（德阳建工，EPC施工）4 阶段: ①施工前=*, ②施工中=按月进度80%支付, ③竣工=支付至90%, ④结算后=支付至97%, ⑤质保=3%尾款

### 2.4 合同月度明细表 `t_contract_monthly_detail`

**来源**: 文件1  
- 合同支付情况 Sheet（Col P~AO，24 个月份列 → 支付金额）  
- 产值情况 (2) Sheet（Col S~AR，24 个月份列 → 产值金额）

**设计思路**: 将横向月份列转为纵向存储（每行=一个合同的一个月份）。

```sql
CREATE TABLE t_contract_monthly_detail (
    id              UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    contract_id     UUID            NOT NULL,                    -- FK → t_contracts
    contract_no     VARCHAR(20)     NOT NULL,                    -- 冗余
    -- 月份
    detail_month    DATE            NOT NULL,                    -- 支付/产值月份（如 2026-01-01）
    -- 支付金额（来自 合同支付情况 Sheet，单位：分。源数据：万元→×1000000）
    payment_amount  BIGINT          DEFAULT 0,                   -- 当月支付金额（分）
    -- 产值金额（来自 产值情况 (2) Sheet，单位：分）
    output_amount   BIGINT          DEFAULT 0,                   -- 当月产值（分）
    -- 产值汇总（来自 产值情况 (2) Sheet，冗余存储便于查询）
    est_cumulative_payable    BIGINT DEFAULT 0,                  -- 预估累计应支付（分）
    actual_cumulative_paid    BIGINT DEFAULT 0,                  -- 实际累计已支付（分）
    est_remaining_unpaid      BIGINT DEFAULT 0,                  -- 预估剩余未支付（分）
    cumulative_completed      BIGINT DEFAULT 0,                  -- 累计已完成产值（分）
    remaining_uncompleted     BIGINT DEFAULT 0,                  -- 剩余未完成产值（分）
    completed_ratio           DECIMAL(8,4),                      -- 已完比例
    remaining_ratio           DECIMAL(8,4),                      -- 剩余未完比例
    -- 时间戳
    created_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW()
);

-- 关键约束
CREATE UNIQUE INDEX idx_contract_monthly_unique
    ON t_contract_monthly_detail(contract_no, detail_month);

COMMENT ON TABLE t_contract_monthly_detail IS '合同逐月支付与产值明细。源数据为横向月份列（Excel 日期序列值45962~46722），导入时转为纵向';
COMMENT ON COLUMN t_contract_monthly_detail.detail_month IS '月份，源数据为 Excel 日期序列值（如45962=2025-11-01）。转换公式: DATE(1899,12,30)+序列值';
```

### 2.5 合同支付记录表 `t_contract_payments`（V2.1 新增）

**来源**: 前端需求文档 二类费用 §7.4 支付台账  
**用途**: 记录每笔实际支付，支撑支付台账、对账报告、逾期判断。与 `t_contract_payment_stages` 的汇总字段（`cumulative_paid`）不同，此表是**逐笔流水**。

```sql
CREATE TABLE t_contract_payments (
    id              UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    contract_id     UUID            NOT NULL,                    -- FK → t_contracts
    contract_no     VARCHAR(20)     NOT NULL,                    -- 冗余
    stage_id        UUID,                                       -- FK → t_contract_payment_stages（可选，关联到具体节点）
    -- 支付信息
    payment_date    DATE            NOT NULL,                    -- 实际支付日期
    payment_amount  BIGINT          NOT NULL DEFAULT 0,          -- 本次支付金额（分）
    payment_voucher VARCHAR(500),                                -- 支付凭证（文件路径或编号）
    payer           VARCHAR(200),                                -- 付款方
    payee           VARCHAR(200),                                -- 收款方
    remark          TEXT,                                        -- 备注
    -- 时间戳
    created_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_contract_payments_date ON t_contract_payments(contract_id, payment_date);
CREATE INDEX idx_contract_payments_no ON t_contract_payments(contract_no);

COMMENT ON TABLE t_contract_payments IS '合同逐笔支付记录。与 t_contract_payment_stages 的关系：stages 存支付条款+汇总金额，payments 存每笔实际支付流水';
COMMENT ON COLUMN t_contract_payments.stage_id IS '关联到支付节点。一笔支付可以对应一个节点（如"竣工验收后支付30%"），也可以不关联';
```

**与 `t_contract_payment_stages` 的关系**:
- `t_contract_payment_stages` = 支付**条款** + 汇总（"合同规定竣工后支付30%，即XXX元"）
- `t_contract_payments` = 支付**流水**（"2026-03-15 实际支付了 XXX 元，凭证号 YYY"）
- 对账时：SUM(payments.amount) GROUP BY contract = stages.cumulative_paid

### 2.6 进度款审核表 `t_progress_payment_review`

**来源**: 文件2（进度款审核汇总表.xls，5 个 Sheet 中的 4 个数据 Sheet）  
**设计思路**: 4 类单位（设备/设计/施工/监理过控）共用一张表，通过 `review_type` 区分。  
- 设备/设计: 9 列通用字段
- 施工: 额外 4 列（建安费+安全文明费+各自付款比例）
- 监理过控: 额外 6 列（监理费+造价咨询费+结算费+各自付款比例）

**金额单位**: 源数据为 **元**，存储为分需 ×100。

```sql
CREATE TABLE t_progress_payment_review (
    id              UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id      UUID            NOT NULL,                    -- FK → t_projects
    contract_id     UUID,                                       -- FK → t_contracts（可选）
    -- 审核标识
    review_type     VARCHAR(20)     NOT NULL,                    -- 设备/设计/施工/监理过控
    period_month    VARCHAR(50)     NOT NULL,                    -- 期数/月份（如"第一期/2026年01月"）
    period_no       INTEGER         NOT NULL DEFAULT 1,          -- 期次编号（1,2,3...）
    applicant_unit  VARCHAR(200)    NOT NULL,                    -- 申请单位
    -- 通用金额字段（所有 review_type 共有，单位：分。源数据：元→×100）
    contract_amount             BIGINT  NOT NULL DEFAULT 0,      -- 合同金额（分）
    cumulative_reported_output  BIGINT  NOT NULL DEFAULT 0,      -- 累计报送产值（分）
    payment_ratio               DECIMAL(6,4),                    -- 付款比例（如 0.2=20%）
    current_audited_output      BIGINT  NOT NULL DEFAULT 0,      -- 本期审核产值（分）
    cumulative_audited_output   BIGINT  NOT NULL DEFAULT 0,      -- 累计审核产值（分）
    cumulative_audited_payable  BIGINT  NOT NULL DEFAULT 0,      -- 累计审核应付款（分）
    -- 施工类特有字段（review_type='施工' 时使用）
    constr_install_fee          BIGINT  DEFAULT 0,               -- 建筑安装工程费（分）
    safety_civil_fee            BIGINT  DEFAULT 0,               -- 安全文明施工费（分）
    constr_install_pay_ratio    DECIMAL(6,4),                    -- 建筑安装工程费付款比例
    safety_civil_pay_ratio      DECIMAL(6,4),                    -- 安全文明施工费付款比例
    -- 监理过控类特有字段（review_type='监理过控' 时使用）
    supervision_fee             BIGINT  DEFAULT 0,               -- 工程监理服务费（分）
    cost_consult_fee            BIGINT  DEFAULT 0,               -- 造价咨询服务费（分）
    settlement_fee              BIGINT  DEFAULT 0,               -- 竣工结算服务费（分）
    supervision_pay_ratio       DECIMAL(6,4),                    -- 工程监理服务费付款比例
    cost_consult_pay_ratio      DECIMAL(6,4),                    -- 造价咨询服务费付款比例
    settlement_pay_ratio        DECIMAL(6,4),                    -- 竣工结算服务费付款比例
    -- 备注
    remark          TEXT,                                       -- 备注（含税说明、分账说明）
    tax_rate        DECIMAL(4,2),                                -- 税率（如 9.00=9%, 6.00=6%）
    -- 审核状态
    audit_status    VARCHAR(20)     NOT NULL DEFAULT 'pending',  -- pending/auditing/audited/paid
    submit_date     DATE,                                        -- 申报日期
    audited_by      UUID,                                        -- FK → t_users
    audited_at      TIMESTAMPTZ,
    -- 时间戳
    created_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW()
);

COMMENT ON TABLE t_progress_payment_review IS '进度款审核汇总表，4类单位（设备/设计/施工/监理过控）共用。金额单位：分';
COMMENT ON COLUMN t_progress_payment_review.review_type IS '审核类型枚举：设备/设计/施工/监理过控（来自 dict_review_type）';
COMMENT ON COLUMN t_progress_payment_review.tax_rate IS '税率：设备/施工单位9%，设计/监理过控单位6%';
```

## 七、数据导入注意事项

1. **文件2（.xls 格式）**: 需先用 Excel 另存为 `.xlsx` 或通过 xlrd 读取后写入数据库
2. **日期序列值**: 文件1 和文件6 的月份标题为 Excel 序列值，导入时转换：`DATE '1899-12-30' + serial_number::INT`
3. **公式错误值**: `#DIV/0!`、`#REF!` → 导入时替换为 SQL NULL
4. **含单位后缀的数字**: `"19320.33万元"` → 提取数值 19320.33，乘以 1000000 转为分
5. **换行拼接**: `"产投集团\n省食油储备"` → 决定是保留还是拆分到关联表
6. **Sheet1 透视表（文件5）**: 跳过不导入
7. **文件5 续投项目的标题为"2025年"**: 确认实际数据年度后再映射 `fund_year`

## 八、与 V1.0 规范的关键变更

| V1.0 | V2.0 | 原因 |
